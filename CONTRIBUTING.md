# Contributing

Contributions are welcome. Please follow these guidelines.

---

## How to Contribute

1. Fork the repository on GitHub.
2. Create a branch: `git checkout -b feature/your-improvement`.
3. Make your changes.
4. Test that the skill still produces correct, complete answers on at least
   two different LeetCode problems across different patterns.
5. Open a pull request with a clear description of what changed and why.

---

## What to Contribute

- New pattern entries in the Pattern Recognition Table.
- Additional C++ pitfall entries.
- Improved visual dry run examples for specific problem types.
- Better interview tip phrasing.
- New similar problem recommendations.
- Bug fixes in the example answers inside SKILL.md or AGENTS.md.

---

## Standards

- All code examples must be valid C++ that would compile under LeetCode's environment.
- All new sections must follow the existing formatting conventions.
- Keep AGENTS.md and SKILL.md consistent — if you add a section rule to AGENTS.md,
  update the Required Answer Sections list in SKILL.md too.
- Update CHANGELOG.md with a summary of your change under a new version block.
